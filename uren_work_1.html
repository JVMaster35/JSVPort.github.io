Student Project: Real-Time Strategy Multiplayer

Here is the project I conducted regarding multiplayer interfaces using Unreal Engine during my 3rd Year of University.

Project Details

Platform

  • Windows

Engine/Language

  • Unreal Engine (Blueprints)

Project

  • Student Assignment
  • 3d, multiplayer, real-time strategy game

About Project

This game was developed as part of my 3rd year Games Technique Course. For this assignment we were tasked to create a virtual environment for a strategy game, which incorporates multiplayer functionality, allowing multiple players, up to ?? in total, to be able to enter the game level and participate in the game.

In terms of the genre of the strategy game this project will be based on, I went for the popular Real-Time Strategy, or RTS, which allows multiplayers to compete against one another at the exact same time, without being divided by turns. This may allow additional players to enter an in-progress game, without interrupting the other players already part of the game. As with any RTS game, players will view and control everything from the view of a top-down camera, which will be used to navigate across the level to monitor both their units and enemy positions.

After much trial and error, while I was able to get the multiplayer functionality of the RTS working, sadly, the game itself was not finished up to professioal standards, as it lacks the profound building placement or unit spawning found is most tactical games. While this maybe due to focusing on developing the game solely through Unreal Engine's blueprints, I feel that, given more time, understanding anf resources, I could have created a fully functional RTS level.
